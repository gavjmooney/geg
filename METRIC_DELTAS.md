## Expected impact on the TVCG / Mooney et al. landscape dataset

The TVCG dataset was produced entirely by *straight-line* layouts (Fruchterman‚ÄìReingold, Kamada‚ÄìKawai, DRGraph, Sugiyama, HOLA, circular, random ‚Äî plus the reapplication pass in the extended dataset). None of those draw edges as cubic/quadratic Bezier curves. Two things to check:

**`aspect_ratio` degenerate case.** Any drawing whose node-position bounding box is exactly 0 in height or width would flip 0 ‚Üí 1 under the refactor. In practice this requires every node to share an x-coordinate (vertical line) or y-coordinate (horizontal line). For the force-directed and stress layouts this is a measure-zero event (continuous distribution); for circular it cannot happen with n ‚â• 3; for HOLA and Sugiyama it would require a trivially small input. Expected impact: a handful of drawings at most, almost certainly 0 in the Mooney et al. corpus. **Worth re-running the Asp column on the dataset to confirm no drawings trip this.**

**`edge_orthogonality` polyline change.** The old public `edge_orthogonality` only consulted node positions ‚Äî it ignored every edge's `path` attribute. The paper's new definition (and the old `curved_edge_orthogonality` helper) uses the full polyline. For straight-line layouts the edge path is a single `M..L..` segment whose orientation equals the node-to-node direction, so the two formulas produce identical output. **Only layouts that emit polyline/curve paths in the TVCG dataset would change** ‚Äî and the TVCG corpus appears to be straight-line only. If Sugiyama or HOLA drawings in the TVCG corpus stored polyline paths (orthogonal layouts often do), those would need rechecking.

**Every other metric** (AR, CA, EC, ELD, EO on straight edges, GR, KSM, NP, NR, NU) is byte-identical on the fixture set between branches. Three of those underwent refactors that were pure internal dedup / disconnected-handling ‚Äî KSM and NP gained a new code path on *disconnected* drawings (weighted sum per paper ¬ß3.3), but the TVCG corpus is connected by construction (layouts applied to single-component graphs). Expected impact on TVCG: none.
